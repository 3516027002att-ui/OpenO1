# OpenO1

OpenO1 研究如何利用充足的推理时间、模型调用和计算资源，提高困难问题的求解上限。

系统通过外层运行时组织多轮推理、搜索、代码执行、符号演算、并行探索和外部验证，并维护可恢复的任务状态。运行过程中保留证据、失败路线、未决问题和验证结果，支持长期推进、分支、回溯与重新分配计算。

项目处于早期阶段，当前以数学推理为首个验证场景。

## 当前工作

- 维护可恢复的任务与推理状态
- 管理解法分支、回溯和计算分配
- 接入代码执行、符号计算和领域验证器
- 记录模型调用、工具结果、失败原因和最终证据
- 对比直接回答与长时间运行后的准确率、稳定性和计算开销

## 实现原则

OpenO1 不采用固定的 Solver-Critic-Judge 或多 Agent 流程。系统根据任务状态增加分支、工具、审阅和验证。

运行时负责保存状态、执行动作、管理分支和批准结束。模型负责提出推导、候选结论和下一步动作。关键结论需要验证，失败路线会被保留，额外调用需要产生新的推导、证据、反例或状态变化。

## 评测

第一阶段评测推理预算与求解质量之间的关系。实验保留模型直接回答的基线，并记录准确率、运行时间、token、工具调用、失败率和运行配置。

目标是把更多计算，真正变成更多智能。
