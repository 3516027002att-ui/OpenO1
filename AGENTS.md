# OpenO1 Agent 协作规则

## 项目定位

OpenO1 是一个通用推理受控状态机项目。数学推理是第一阶段验证场景，但项目目标不是只做数学题，而是沉淀一套可审阅、可验证、可回滚、可迭代的通用推理框架。

## 顶层规则

- 中心引擎是唯一状态所有者。
- Agent 只能提交结构化中间结果，不能直接改写任务目标、共享最终结论或最终答案。
- 所有结论必须带来源、证据、假设、风险和验证状态。
- 验证失败、预算耗尽、冲突未解时，必须报告阻断原因，不能合成伪成功答案。
- 所有“逼近 gpto1”相关描述都必须表达为目标或实验假设，不能表达为已达成事实。
- 先功能闭环，后性能优化。

## 外部项目学习记录规则

之后在 GPT 项目页面中的任何聊天里，只要讨论或分析了外部项目、论文、框架、工具链，并且发现对 OpenO1 有借鉴价值，就应该优先沉淀到：

`docs/external-project-lessons.md`

记录时必须区分：

- 外部项目已经明确实现的事实。
- 对 OpenO1 的迁移判断。
- 仍需实验验证的假设。

不得把外部项目宣传口径直接当成 OpenO1 的结论。尤其是涉及推理能力、benchmark、数学能力、o1 风格能力、heavy model 行为时，必须保持“目标、假设、实验结果”三者分离。
