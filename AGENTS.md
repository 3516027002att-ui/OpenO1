## 1. 项目愿景与目标
- **项目定位**：OpenO1 是一个开放、模型无关的 Agentic Reasoning Runtime。
- **核心命题**：模型在单次调用中的表现，不等于它在长期运行系统中的最终能力。OpenO1 将工具使用、长期行动、并行探索和环境反馈组织成外部化推理过程。
- **终极目标**：把模型的 agentic capability 转化为 reasoning capability，使开源模型能够用额外时间、行动和反馈扩大问题解决上限，持续压缩与最前沿推理系统之间的差距。
- **短期目标**：聚焦数学推理，基于本地开源模型，验证规划、路线探索、工具调用、外部验证、回溯和预算控制对推理能力的实际增益。
- **底线红线**：不承诺虚高能力；不用关闭校验、跳过审阅、吞掉失败来制造表面成功；一切进展必须基于可复现的实验和 benchmark。

## 2. 架构规范（中心引擎与模块）
中心引擎是唯一状态所有者与任务编排者，负责调度子模块并管理 `Shared Reasoning State`。

- **状态内容**：至少记录任务目标、事实、命题、假设、证据、反例、冲突、未决问题、失败路线、风险、预算和候选结论。
- **子模块约束**：不能直接改写任务目标、共享结论或最终答案，只能向中心引擎提交结构化中间结果。
- **按需调度**：不能把固定 Solver–Critic–Judge、多 Agent 辩论或机械反思当成默认仪式。中心引擎应根据问题结构、验证需求和预期边际收益选择 Agent、工具与路线数量。
- **独立性要求**：多 Agent 的价值来自不同假设、证据来源、搜索空间或错误分布；仅更换角色名称不构成有效独立探索。
- **行动语义**：搜索、代码执行、符号计算、实验和文件操作必须服务于推理，产生可记录的证据、反例、状态变化或验证结果。
- **核心模块职责**：
  - **Reasoning Planner**：构造研究计划，维护未决问题，提出可验证的候选路线。
  - **AgentTeam**：按需探索相互独立的假设、解法、反例和验证路径。
  - **Domain Verifier**：验证特定领域中的关键步骤与最终结论，并明确工具结果的适用边界。
  - **Review Gate**：确认目标已达、关键命题有支持、无未处理高危阻断项后才允许输出；失败时退回继续推理或报告根本原因。
  - **Budget Manager / Stop Controller**：根据剩余预算、进展速度和继续推理的预期收益决定继续、分叉、回溯、降级或终止。
  - **Benchmark / Eval**：记录模型、配置、运行轨迹及 AIME/MATH/GSM8K 表现，分离基础模型能力与 runtime 增益。

## 3. 协作与质量协议
- **数学质量要求**：
  - 变量定义一致，无逻辑或代数跳步，必须检查定义域、边界值与特殊值。
  - 严禁将“看起来合理”视作证明、隐瞒关键推导错误、使用 benchmark 外的散点样例宣称整体能力。
- **证据要求**：
  - 工具结果属于带边界的证据，不能自动视为真理。
  - 数值测试可以发现反例，不能替代完整证明。
  - 被否定路线、失败工具结果和重复错误必须写入状态，避免系统反复消耗计算。
- **演化要求**：
  - Runtime 应随基础模型能力变化而简化或扩展。
  - 当模型已经内化某项流程时，应优先删除冗余 harness，而不是维护永久仪式。

## 5. 当前阶段默认假设
- 当前阶段以规范、最小中心引擎和可复现实验骨架为主。
- 短期目标以 AIME、MATH、GSM8K 作为核心验收题集。
- 每项实验必须保留模型直接回答 baseline，并记录准确率、token、时间、工具调用、失败率和运行配置。
- 所有“压缩与前沿模型差距”的描述只能作为实验假设或目标，不可作为已达成事实。
- 第一阶段优先研究适合外部验证的推理任务，随后扩展到代码推理、形式证明、科学计算和研究分析。

## 6. 框架规范

主体引擎的功能：

1. 接收任务目标、已知条件、目标结论、格式、约束与成功标准。
2. 分析任务类型、难度、可验证性和候选研究空间，判断是否需要工具、外部证据、路线分叉或多 Agent 协作。
3. 从单路线低成本尝试开始；仅在存在明确收益预期时创建 AgentTeam、增加采样、调用验证器或扩展预算。
4. 将模型输出解析为结构化动作，由中心引擎执行工具、更新状态、管理路线和记录证据。
5. 在路线失败、证据冲突或前提被推翻时支持回溯，不允许只沿原路线重复修补。
6. 由 Review Gate 与 Stop Controller 决定继续、分叉、回滚、输出最佳努力结果或终止。
7. 保存完整运行轨迹，为后续 benchmark、策略改进和模型能力分析提供可复现依据。

## 7. 一句话原则

> **Turn agentic capability into reasoning capability.**
>
> 把会行动的模型，组织成会长时间思考的系统。
